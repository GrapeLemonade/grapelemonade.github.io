Chapter 4C GKR 协议

基本定义

GKR 协议是具有较好泛用性的协议。GKR 协议能证明满足一定限制的算术电路 C\mathcal{C}C 的输出。

对于 C={Cn∣n∈N}\mathcal{C}=\{\mathcal{C}_n\mid n\in \mathbb{N}\}C={Cn​∣n∈N} 的要求：

  • C\mathcal{C}C 是 log-space uniform 的，这指的是存在一个运行在对数空间内的图灵机 MMM，其对于输入串 1n1^n1n 的输出是 Cn\mathcal{C}_nCn​ 的一个描述。
  • C\mathcal{C}C 中所有门至多接受 2 个输入。

设 C\mathcal{C}C 中门的数量为 SSS，深度为 ddd。则使用 GKR 协议证明 C\mathcal{C}C 的输出可以做到：

  • V\mathcal{V}V 运行时间：O(dlog⁡S)\mathcal{O}(d\log S)O(dlogS)
  • P\mathcal{P}P 运行时间：O(poly(S))\mathcal{O}(\mathrm{poly}(S))O(poly(S))，可以优化到 O(S)\mathcal{O}(S)O(S)

协议结构

P\mathcal{P}P 与 V\mathcal{V}V 约定证明 C\mathcal{C}C 在 F\mathbb{F}F 上的输。C\mathcal{C}C 满足上面要求的性质，并且 C\mathcal{C}C 是分层电路，每层的所有门均只与相邻层的门相连。设 C\mathcal{C}C 的深度为 ddd。

将非分层电路转化为分层电路的影响至多 ddd 倍。

将层自顶向下从 000 到 ddd 编号，第 000 层包含所有输出，第 ddd 层包含所有输入。

可靠性误差

Sum-Check 本身的 soundness error 是 O((∑deg⁡g)/∣F∣)\mathcal{O}((\sum \deg g)/|\mathbb{F}|)O((∑degg)/∣F∣) 的，而 GKR 协议中所有应用 Sum-Check 协议的多项式对于其中一个未定元的次数均为常数，因此 Sum-Check 引入的 soundness error 是 O((dlog⁡S)/∣F∣)\mathcal{O}((d\log S)/|\mathbb{F}|)O((dlogS)/∣F∣) 的。

多点合一引入的 soundness error 不超过 ((m−1)log⁡n)/∣F∣((m-1)\log n)/|\mathbb{F}|((m−1)logn)/∣F∣，其中 mmm 为点数，log⁡n\log nlogn 是维数。因此每轮的误差不超过 ki/∣F∣k_i/|\mathbb{F}|ki​/∣F∣，总误差不超过 (dlog⁡S)/∣F∣(d\log S)/|\mathbb{F}|(dlogS)/∣F∣。

因此 GKR 协议的总 soundness error 是 O((dlog⁡S)/∣F∣)\mathcal{O}((d\log S)/|\mathbb{F}|)O((dlogS)/∣F∣) 的。

结构 MLE 求值效率问题

求 add~i,mult~i\widetilde{\mathrm{add}}_i, \widetilde{\mathrm{mult}}_iaddi​,multi​ （下面亦称结构 MLE）的随机点值并不容易，注意这里的点是 Fki+2ki+1\mathbb{F}^{k_i+2k_{i+1}}Fki​+2ki+1​ 中的元素。书的作者罗列了部分进展：

  • 对于矩阵乘法、模式匹配、FFT、一系列流式算法，对应的算术电路 Cn\mathcal{C}_nCn​ 对应的 add~i,mult~i\widetilde{\mathrm{add}}_i, \widetilde{\mathrm{mult}}_iaddi​,multi​ 可在 O(ki+ki+1)\mathcal{O}(k_i+k_{i+1})O(ki​+ki+1​) 时间内求点值。
  • 只要 addi,multi\mathrm{add}_i, \mathrm{mult}_iaddi​,multi​ 可在 “read-once branching programs” 模型下计算，则求点值可在对数时间内完成。
  • 只要 C\mathcal{C}C 是 log-space uniform 的，V\mathcal{V}V 总可以在 O(log⁡n)\mathcal{O}(\log n)O(logn) 空间内求得点值。由于 ri,b∗,c∗r_i, b^\ast, c^\astri​,b∗,c∗ 的确定实际上与电路及输入无关，只与电路的规模有关，而结构 MLE 只与电路结构有关，实际上这些点值可以离线处理，而不必在证明时在线计算。

GKR 协议的原作者采取了另一种技术路线。他们给出了一个证明非确定性图灵机在对数空间内可求解的问题的方案，并证明了其对应的结构 MLE 可在对数时间内求点值。对于任意 log-space uniform 的电路，目前无法判断其结构 MLE 是否可在 O(polylog⁡S)\mathcal{O}(\mathrm{poly} \log S)O(polylogS) 时间内求点值，但由于其结构本身可在对数空间内生成，可对求其结构 MLE 的点值本身应用他们提出的协议，如此得到所有 log-space uniform 的电路均可高效验证。

可以使用密码学中的多项式承诺方案论证 P\mathcal{P}P 给出的结构 MLE 的点值的正确性。在这种情况下证明系统弱化为论证系统（argument system）。
