由于这里 r1,r2r_1, r_2r1​,r2​ 是 Flog⁡n\mathbb{F}^{\log n}Flogn 中的向量，证明 f~A2(r1,r2)\tilde f_{A^2}(r_1, r_2)f~​A2​(r1​,r2​) 的值需要花费双方各 O(n2)\mathcal{O}(n^2)O(n2) 时间。

分析

  • 通信量：整体 Sum-Check 需要 2log⁡n2\log n2logn 轮，每轮多项式的次数至多 222；内层 MatMul IP 需要 log⁡n\log nlogn 轮，每轮同样至多 222 次。因此总通信量是 9log⁡n9\log n9logn，轮次 3log⁡n3\log n3logn。
  • V\mathcal{V}V：查谕示机是主要开销，共 O(n2)\mathcal{O}(n^2)O(n2)。
  • P\mathcal{P}P：P\mathcal{P}P 已经知道 A2\bm{A}^2A2，额外证明开销 O(n2)\mathcal{O}(n^2)O(n2)。

相比于直接使用 Sum-Check 协议，优势在于 P\mathcal{P}P 的运行效率。

多点证明优化

直接使用 Sum-Check 协议证明三元环个数时，最终 V\mathcal{V}V 需要求 f~A\tilde f_Af~​A​ 的三个点值以完成查 g=∑∏cycf~A(i,j)g=\sum \prod_{cyc} \tilde f_A(i, j)g=∑∏cyc​f~​A​(i,j) 的谕示机的任务。

这是一个常见需求，为求 ggg 的一个点值需要求某个 MLE W~\tilde WW~ 的多个点值。

设 W~:Flog⁡n→F\tilde W:\mathbb{F}^{\log n}\to \mathbb{F}W~:Flogn→F 是某个 (log⁡n)(\log n)(logn)-元函数的 MLE，考虑求 W~(b),W~(c)\tilde W(b), \tilde W(c)W~(b),W~(c)。

可以通过一轮交互将两次求值减少为一次 W~(r)\tilde W(r)W~(r)。这指的是由 P\mathcal{P}P 直接给出 W~(b),W~(c)\tilde W(b), \tilde W(c)W~(b),W~(c) 的值，V\mathcal{V}V 可通过查谕示机求 W~(r)\tilde W(r)W~(r) 验证上面两个点值的正确性。

方案

设 ℓ:F→Flog⁡n\ell:\mathbb{F}\to \mathbb{F}^{\log n}ℓ:F→Flogn 为空间 V=Flog⁡nV=\mathbb{F}^{\log n}V=Flogn 中过 b,cb, cb,c 的直线，不妨设 ℓ(0)=b,ℓ(1)=c\ell(0)=b, \ell(1)=cℓ(0)=b,ℓ(1)=c。容易证明满足条件的 ℓ\ellℓ 唯一。（实际上显然有 ℓ(x)=b+x(c−b)\ell(x)=b+x(c-b)ℓ(x)=b+x(c−b)。）

P\mathcal{P}P 发送多项式 qqq，声称 q=W~∘ℓq=\tilde W\circ \ellq=W~∘ℓ，即函数 W~\tilde WW~ 将定义域限制在 ℓ\ellℓ 上所得的多项式。注意这里 q:F→Fq:\mathbb{F}\to \mathbb{F}q:F→F，且 deg⁡q≤deg⁡W~⋅deg⁡ℓ<log⁡n\deg q\le \deg \tilde W\cdot \deg \ell < \log ndegq≤degW~⋅degℓ<logn。

此时对于 V\mathcal{V}V 而言，q(0),q(1)q(0), q(1)q(0),q(1) 即为 P\mathcal{P}P 声称的 W~(b),W~(c)\tilde W(b), \tilde W(c)W~(b),W~(c) 的值。V\mathcal{V}V 随机选取 r∗∈Fr^\ast \in \mathbb{F}r∗∈F，q(r∗)q(r^\ast)q(r∗) 即为 P\mathcal{P}P 声称的 W~(r∗)\tilde W(r^\ast)W~(r∗) 的值。V\mathcal{V}V 查谕示机验证 q(r∗)=W~(r∗)q(r^\ast)=\tilde W(r^\ast)q(r∗)=W~(r∗) 即可。

分析

  • completeness：显然对于所有正确的 q=W~∘ℓq=\tilde W\circ \ellq=W~∘ℓ，V\mathcal{V}V 都以概率 111 接受证明。
  • soundness：若 q≠W~∘ℓq\neq \tilde W\circ \ellq=W~∘ℓ，明显等式两端均为 F\mathbb{F}F 上的度至多 log⁡n\log nlogn 的多项式，因此随机选取 r∗∈Fr^\ast \in \mathbb{F}r∗∈F 时，soundness error 不超过 (log⁡n)/∣F∣(\log n)/\left|\mathbb{F}\right|(logn)/∣F∣。

显然对于 mmm 个点依然可以使用此方案。

矩阵快速幂

给定 F\mathbb{F}F 上的 nnn 阶方阵 A\bm{A}A，希望证明 (Ak)xy(\bm{A}^k)_{xy}(Ak)xy​，以下假设 kkk 是 222 的幂。则有

(Ak)xy=∑z=1n(Ak/2)xz(Ak/2)zy(\bm{A}^k)_{xy}=\sum\limits_{z=1}^{n} (\bm{A}^{k/2})_{xz}(\bm{A}^{k/2})_{zy} (Ak)xy​=z=1∑n​(Ak/2)xz​(Ak/2)zy​

这明显可以用 MatMul IP 解决：

  • 通信轮次：log⁡n\log nlogn
  • 通信量：3log⁡n3\log n3logn
  • V\mathcal{V}V 运行时间：T+O(log⁡n)T+\mathcal{O}(\log n)T+O(logn)
  • P\mathcal{P}P 运行时间：O(n2)\mathcal{O}(n^2)O(n2)

记 (x′,y′)=ℓ(r∗)(x', y')=\ell(r^\ast)(x′,y′)=ℓ(r∗)，即此时 V\mathcal{V}V 需要验证 f~Ak/2(x′,y′)\tilde f_{A^{k/2}}(x', y')f~​Ak/2​(x′,y′) 的值，而

MatMul + GKR

使用 MatMul IP 结合 GKR 协议可以获得更广的应用范围。

考虑求有向图 GGG 的直径。令其邻接矩阵为 A\bm{A}A。直径 ddd 是使得 (A+I)d(\bm{A}+\bm{I})^d(A+I)d 中所有元素非零的最小的正整数。

证明直径需要证明两个事实：

  1. (A+I)d−1(\bm{A}+\bm{I})^{d-1}(A+I)d−1 中存在一个元素为零
  2. D≔(A+I)d\bm{D}\coloneqq(\bm{A}+\bm{I})^dD:=(A+I)d 中所有元素非零

第一个事实的证明很容易，P\mathcal{P}P 发送下标 i,ji, ji,j 即可，V\mathcal{V}V 选取一个域 ∣F∣\left|\mathbb{F}\right|∣F∣，接下来使用 MatMul IP 即可。

第二个事实中需要统计矩阵中非零元素的数量，这由 GKR 协议完成，其中最后一步需要证明 f~D\tilde f_{D}f~​D​ 的一个点值，这可以使用 MatMul IP 完成。

书的作者在这里直接给出了结果，V\mathcal{V}V 的运行时间是 O(mlog⁡n)\mathcal{O}(m\log n)O(mlogn)，通信量 O(polylog⁡n)\mathcal{O}(\mathrm{poly}\log n)O(polylogn)。

再探 IP = PSPACE

这部分只是一个 sketch，并不涉及具体细节。

设 MMM 是一个图灵机，其输入长 mmm 位，运行空间不超过 sss 位。

考虑其运行状态（由本身状态与纸带情况组成）的总数，显然至多 O(2s)\mathcal{O}(2^s)O(2s)。（感觉图灵机的状态本身也要乘上去，不过这里暂且假设图灵机的状态数量是 O(s)\mathcal{O}(s)O(s) 的应该没问题 = =）

那么对于 N=O(2s)N=\mathcal{O}(2^s)N=O(2s)，可将 MMM 的状态转移表示为 NNN 阶方阵 A\bm{A}A。

不失一般性，可设初始状态与接受状态唯一，这样对于给定输入，判断 MMM 对该输入是否接受，可转化为求 AN\bm{A}^{N}AN 中的某一项是否为零。

使用上面介绍的矩阵快速幂方法即可解决。

  • 通信轮次：O(slog⁡N)\mathcal{O}(s\log N)O(slogN)
  • V\mathcal{V}V 运行时间：查谕示机是最耗时的一步。图灵机的性质使得 A\bm{A}A 高度稀疏，充分利用后可以达到 O(s⋅m)\mathcal{O}(s\cdot m)O(s⋅m) 时间。总时间 O(slog⁡N+m⋅s)\mathcal{O}(s\log N+m\cdot s)O(slogN+m⋅s)。
  • P\mathcal{P}P 运行时间：O(poly(n))\mathcal{O}(\mathrm{poly}(n))O(poly(n))。

由此可推出 PSPACE⊆IP\mathsf{PSPACE}\subseteq\mathsf{IP}PSPACE⊆IP。
