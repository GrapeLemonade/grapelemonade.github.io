  3. O(n)O(\sqrt n)O(n​) 暴力分解。
  4. 枚举。
  5. 求 LCM 即可。
  6. 这两个和的闭形式都是耳熟能详的，可以直接手算。
  7. 简单筛法。
  8. 遍历。这里可以每次只减开头加末尾。
  9. 枚举。
  10. 简单筛法。

11-20

  1. 遍历。
  2. 枚举。
  3. 拿 python 直接加。
  4. 直接模拟即可。
  5. 显然 n×mn\times mn×m 方格的答案是 (nm)\binom{n}{m}(mn​)。

21-30

  1. 不多，可以直接枚举。
  2. 直接模拟。
  3. 不多，先预处理出可用的数然后直接使用 bitset 即可。
  4. 有个东西叫 Cantor Expansion，不过感觉比较显然，直接考虑后面的方案数然后拆分就行了。
  5. 这个不多，直接算也行，拿通项搞也行。我们知道 ln⁡10/log⁡φ≈4.784972\ln 10/\log \varphi\approx 4.784972ln10/logφ≈4.784972，因此答案一定在 478547854785 左右。
  6. 这个本质上就是算 101010 在模 d/2s5td/2^s5^td/2s5t 下的阶，其中 2s∥d,5t∥d2^s\parallel d, 5^t\parallel d2s∥d,5t∥d。这里 ∥\parallel∥ 表示恰整除。
  7. 显然对于 ∣n∣≤1000|n|\le 1000∣n∣≤1000，我们知道显然 ∣n2+an+b∣≤2.001×106|n^2+an+b|\le 2.001\times 10^6∣n2+an+b∣≤2.001×106，因此我们可以预处理这个范围内的数是否为素数，之后便可在常数时间内检验素性。我们可以猜想连续为素数的个数长的不是很多，因此可以逐个计算。显然 b2+ab+b=b(b+a+1)b^2+ab+b=b(b+a+1)b2+ab+b=b(b+a+1)，我们知道长度至多为 bbb。
  8. 这里显然是一个 1+∑(4i2−6i+x)1+\sum (4i^2-6i+x)1+∑(4i2−6i+x) 的形式，随便整整或者模拟一下均可。
  9. 直接拿 python 开个 set 解决了。
  10. 枚举。

31-40

  1. 简单 DP。
  2. 不难发现左边的位数加起来必定大于等于右边的位数，因此乘积最大只能是 4 位数。因此我们只需要枚举所有 4 位数及它们的约数即可。
  3. 枚举。
  4. 对于一个 nnn 位数 x=dndn−1…d1‾x=\overline{d_nd_{n-1}\dots d_1}x=dn​dn−1​…d1​​，显然 ∑di10i−1≥10n−1≥n⋅9!≥∑di!\sum d_i10^{i-1}\ge 10^{n-1}\ge n\cdot 9!\ge \sum d_i!∑di​10i−1≥10n−1≥n⋅9!≥∑di​! 当 10n−1≥n⋅9!10^{n-1}\ge n\cdot 9!10n−1≥n⋅9! 时恒成立。不难发现后者等价于 n≥8n\ge 8n≥8。而第一个 ≥\ge≥ 中的等号只有 dn=1d_n=1dn​=1 时取到，而显然 100000000100000000100000000 不满足条件，因此我们只需要枚举所有 7 位数即可。
  5. 如果这个数的位数大于 111，显然只能由 1,3,5,71,3,5,71,3,5,7 构成，这样候选数量就只剩差不多 47/3≈55004^7/3\approx 550047/3≈5500 个了，直接枚举即可。
  6. 小于 10610^6106 的回文数显然不超过 10310^3103 个，直接枚举即可。
  7. 同上题分析，它只能以 2,3,5,72, 3, 5, 72,3,5,7 中的数字开头，中间的数字只能出现 1,3,7,91, 3, 7, 91,3,7,9。由于答案有限且数量较小我们从头开始 DFS 即可。
  8. 起始数字至多不超过 10410^4104，枚举即可。
  9. 考虑枚举 ppp 与最短的边 aaa，不难发现 a+b+c=pa+b+c=pa+b+c=p 与 a2+b2=c2a^2+b^2=c^2a2+b2=c2 限制了 (b,c)(b, c)(b,c) 解数有限。因此直接枚举 (p,a)(p, a)(p,a) 解出 (b,c)(b, c)(b,c) 验证即可。
  10. 直接模拟或者通过分析位数直接计算出这些位均可。
